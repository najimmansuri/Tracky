Honestly, this started as a frustration. Our team tracks video production status in a Google Sheet — WIP, In Review, Done, YTD — and checking it every time felt like opening a spreadsheet just to see if anything changed.

So I thought: what if I could just open a clean dashboard on my phone, see exactly where every video stands, tap one to watch it or read its notes — no spreadsheet hunting? That's Tracky. And it took one afternoon to build, with a lot of help from Claude and Gemini.

The stack (spoiler: it's mostly glue)

No framework. No backend. Just a Google Sheet, a GitHub repo, and some clever automation stitched together.

The flow is simple: Google Sheets is the source of truth. An Apps Script runs every hour, reads the sheet, builds a data.json file, and pushes it to GitHub via the API. GitHub Pages serves a static HTML dashboard that fetches that JSON on load. No server, no database, no cost.

"The whole thing auto-updates every hour. Edit the sheet, and within 60 minutes the dashboard reflects it — everywhere, on any device."

What Tracky actually does

  • Live status counts — WIP, In Review, Done, YTD — updated automatically from the sheet
  • Per-person view so you can see exactly what Nazim or Vishnu is working on at a glance
  • Tap any video card to open a detail sheet with notes, deadline, owner, and a Watch Video button
  • Direct links to YouTube or Figma review — one tap to open the actual video
  • Swipe down to dismiss, tab bar navigation, works perfectly on mobile
  • Overdue and deadline warnings — red for late, amber for due soon

How Claude helped build it

I didn't write much of this code myself. Claude (Anthropic's AI) helped with almost everything — debugging the Apps Script sync logic, fixing a SHA mismatch error that was causing GitHub pushes to fail with a 409, building the entire HTML dashboard UI, and iterating on the design multiple times until it felt right.

The trickiest part was extracting real hyperlinks from Google Sheets cells. When Sheets auto-converts a YouTube URL into a "chip" (a smart linked label), the display text and the actual URL get separated. Claude walked through multiple approaches — getRichTextValues(), the Sheets API v4 with hyperlink fields — to try and get those URLs out. A genuine problem that required real debugging, not just copy-paste.

"Claude didn't just write code — it explained why each approach worked or didn't, which made the whole process feel like pair programming."

Gemini for the style

While Claude handled the logic and structure, I used Gemini to help think through the visual direction and styling choices — the warm sand tones, the Syne + JetBrains Mono font pairing, the overall feel of the dashboard. Having two AI perspectives on different parts of the project was genuinely useful — one for engineering, one for aesthetics.

Was it worth it?

100%. Before this, checking video status meant opening a shared Google Sheet, scrolling, filtering. Now it's a bookmark on my phone. The team can check it from anywhere — a meeting, on the go, without needing sheet access.

And honestly, it was just fun. Building something real and useful in an afternoon, shipping it on GitHub Pages, watching the dashboard update automatically when someone edits the sheet — that feedback loop is satisfying in a way that spreadsheets never are.

If your team tracks anything in Google Sheets and you want a nicer way to view it on mobile — this whole pattern is reusable. The Apps Script + GitHub Pages combo is surprisingly powerful for zero-cost internal tooling.
